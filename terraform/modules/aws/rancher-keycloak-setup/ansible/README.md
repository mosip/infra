# Ansible Playbooks for Rancher and Keycloak Setup

This directory contains Ansible playbooks used by Terraform to deploy Rancher UI and Keycloak on an RKE2 cluster.

## Files

- **install-rancher.yml**: Installs Rancher UI with Helm
- **install-keycloak.yml**: Installs Keycloak using k8s-infra scripts
- **get-rancher-info.yml**: Retrieves Rancher status and URLs
- **get-keycloak-info.yml**: Retrieves Keycloak status and URLs
- **inventory.tpl**: Template for Ansible inventory file (generated by Terraform)

## How It Works

Terraform automatically:
1. Creates an SSH private key file (`.ssh_key`) from the variable
2. Generates an inventory file (`inventory.ini`) with the control plane IP
3. Runs the playbooks using `ansible-playbook` via local-exec provisioner

## Requirements

- Ansible must be installed on the machine running Terraform
- Install with: `pip install ansible` or `apt-get install ansible`

## Variables Passed by Terraform

- `cluster_name`: Name of the RKE2 cluster
- `rancher_hostname_var`: Hostname for Rancher UI
- `rancher_password`: Bootstrap password for Rancher
- `keycloak_hostname_var`: Hostname for Keycloak
- `k8s_infra_repo_url`: Git repository URL for k8s-infra
- `k8s_infra_branch_name`: Branch name for k8s-infra

## Manual Execution (for testing)

If you need to run the playbooks manually:

```bash
# Set up inventory
cat > inventory.ini << EOF
[control_plane]
10.0.3.121 ansible_user=ubuntu ansible_ssh_private_key_file=~/.ssh/mosip-aws.pem

[control_plane:vars]
ansible_ssh_common_args='-o StrictHostKeyChecking=no'
EOF

# Run Rancher installation
ansible-playbook -i inventory.ini install-rancher.yml \
  -e cluster_name='obs4' \
  -e rancher_hostname_var='rancher.obs4.mosip.net' \
  -e rancher_password='admin'

# Run Keycloak installation
ansible-playbook -i inventory.ini install-keycloak.yml \
  -e cluster_name='obs4' \
  -e keycloak_hostname_var='iam.obs4.mosip.net' \
  -e k8s_infra_repo_url='https://github.com/bhumi46/k8s-infra' \
  -e k8s_infra_branch_name='develop'
```

## Benefits Over remote-exec

1. **No Output Suppression**: All output is visible (no sensitive value hiding)
2. **Better Error Messages**: Clear error reporting from Ansible
3. **Idempotent**: Ansible modules are naturally idempotent
4. **Reusable**: Playbooks can be run independently of Terraform
5. **CI/CD Friendly**: Works in GitHub Actions without SSH key issues
